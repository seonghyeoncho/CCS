ORM은 객체지향 프로그래밍과 관계형 데이터베이스 사이의 호환되지 않는 데이터를 변환하는 프로그래밍 기법이다.

ORM 장점
  1. 개발의 생산성 향상
  2. 선언문, 할당, 종료 등과 같은 부수적인 코드가 사라짐으로써 개발의 생상성 향상
  3. 명확한 쿼리
  4. 코드가 간결하고 직관적이기 때문에 가독성 향상
  5. Db 의존성 감소
단점
  1. 튜닝의 어려움
  2. 미세한 수정 및 디버깅의 상대적 어려움
  3. 복잡한 쿼리 작성
  4. 지정된 명령만 수행 가능
  5. DB에 직접 내리는 명령이 아니라서 raw에 비해 느림
